Cómo escribir código autodocumentado

Por David Alonso • 10 Dic, 2025 • 5 min lectura

A menudo escuchamos que "un buen código no necesita comentarios". Aunque esto es una simplificación exagerada, encierra una verdad fundamental: nuestro código debería expresar su intención de la manera más clara posible a través de su propia estructura y nomenclatura.

El poder del Naming

Nombrar variables y funciones es una de las tareas más difíciles y cruciales en programación. Un buen nombre ahorra horas de descifrado mental.

"Cualquier tonto puede escribir código que una computadora entienda. Los buenos programadores escriben código que los humanos entiendan." — Martin Fowler

The Power of Naming

Naming variables and functions is one of the hardest and most critical tasks in programming. A good name saves hours of mental deciphering.

"Any fool can write code that a computer can understand. Good programmers write code that humans can understand." — Martin Fowler

Bad Examples vs. Good Examples

Let's look at a classic example of confusing code:

// Bad
const d = 10;
function calc(a) {
    return a * d; 
}

Now, let's look at the refactored version:

const DAYS_TO_PROCESS = 10;

function calculateTotalProcessingTime(tasksCount) {
    return tasksCount * DAYS_TO_PROCESS;
}

La diferencia es abismal. En el segundo ejemplo, la intención es obvia sin necesidad de una sola línea de comentarios.

Principios Clave

  • Evita números mágicos: Usa constantes con nombres descriptivos.
  • Funciones pequeñas: Una función debería hacer una sola cosa y su nombre debería reflejarlo.
  • Tipado fuerte (si es posible): TypeScript o lenguajes tipados ayudan a documentar las estructuras de datos.

Conclusión

Escribir código legible es un acto de empatía hacia tu yo del futuro y hacia tus compañeros de equipo. Tómate el tiempo de nombrar bien tus variables; es una inversión que siempre paga dividendos.

The difference is abysmal. In the second example, the intent is obvious without needing a single line of comments.

Key Principles

  • Avoid magic numbers: Use constants with descriptive names.
  • Small functions: A function should do one thing, and its name should reflect that.
  • Strong typing (if possible): TypeScript or typed languages help document data structures.

Conclusion

Writing readable code is an act of empathy towards your future self and your teammates. Take the time to name your variables well; it's an investment that always pays dividends.
